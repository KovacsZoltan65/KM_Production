# i18n fejlesztői útmutató

Ez az útmutató a KM_Production lokalizációs szabályait foglalja össze röviden. Célja, hogy az új UI és backend üzenetek ugyanazt a kulcsalapú fordítási rendszert használják.

## Alapelv

Fordítás kizárólag ezekben a fájlokban lehet:

- `lang/en.json`
- `lang/hu.json`

Ne hozz létre PHP fordítási fájlokat, például `lang/en/*.php` vagy `lang/hu/*.php` fájlokat.
Ne hozz létre modulonkénti translation fájlokat.
A két JSON kulcskészlete mindig maradjon szinkronban.

## Kötelező ellenőrzések

Új lokalizációs módosítás után futtasd:

```bash
npm run i18n:check
npm run build
```

A teljes `php artisan test` jelenleg ismerten bukhat egy fókuszon kívüli `CapacityPlanningTest` miatt, ezért ezt külön kell kezelni.

## Kulcsnév szabályok

Használd ezeket a namespace-eket:

```text
actions.*
common.*
fields.*
filters.*
datatable.*
messages.*
confirm.*
status.*
enum.*
navigation.*

auth.*
profile.*
master_data.*
inventory.*
production.*
procurement.*
documents.*
reports.*
intelligence.*
quality.*
```

## Mikor használjunk közös kulcsot?

Közös kulcsot használj:

- gombokra
- mezőnevekre
- általános státuszokra
- táblázat üres és loading szövegekre
- általános confirm és toast üzenetekre

Példák:

```json
"actions.save": "Save",
"fields.name": "Name",
"datatable.empty": "No records found",
"messages.saved": "Saved successfully."
```

## Mikor használjunk modul-specifikus kulcsot?

Modul-specifikus kulcsot használj:

- oldalcímekre
- üzleti fogalmakra
- modulon belüli speciális actionökre
- show/detail nézetek szövegeire

Példák:

```json
"production.tasks.title": "Production tasks",
"procurement.goods_receipts.title": "Goods receipts",
"documents.version_history.title": "Version history"
```

## Mit ne csináljunk?

- Ne használj hardcoded UI szöveget Vue komponensekben.
- Ne tegyél új fordítást PHP translation fájlba.
- Ne közösíts agresszíven modul-specifikus üzleti szövegeket.
- Ne módosítsd csak az egyik JSON fájlt.
- Ne törölj `status.*` vagy `enum.*` kulcsokat csak azért, mert statikus kereséssel nem látszanak használtnak; ezek gyakran dinamikusan hivatkozott kulcsok.

## Vue használati példák

Template-ben:

```vue
{{ $t('actions.save') }}
```

Script setupban:

```js
import { trans } from 'laravel-vue-i18n'

const title = trans('production.tasks.title')
```

Konfigurációs objektumokban:

```js
const columns = [
    { field: 'name', headerKey: 'fields.name' },
]
```

## Backend használati példák

```php
__('messages.saved')
__('production.tasks.messages.started')
```

## Végső ellenőrzés

Commit előtt futtasd:

```bash
npm run i18n:check
npm run build
```

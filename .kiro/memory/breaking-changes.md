# Breaking Changes

## Purpose

This permanent memory file records changes that break compatibility, workflows, data assumptions, or documentation navigation.

## How to Record Entries

Record breaking changes when users, developers, AI agents, integrations, tests, deployments, or documentation links must adapt.

## Template

```md
## YYYY-MM-DD - Short title

Context:

Breaking Change:

Affected Areas:

Migration or Adaptation:

Future AI Guidance:

Related:
```

## Examples

```md
## 2026-06-28 - Documentation navigation moved to `.kiro/index.md`

Context: AI-native documentation consolidation.

Breaking Change: AI agents should no longer rely on AGENTS.md as the only documentation map.

Affected Areas: Agent onboarding and documentation discovery.

Migration or Adaptation: Start with `AGENTS.md`, then use `.kiro/index.md` for the full layer map.

Future AI Guidance: Update references when documentation entry points change.

Related: `AGENTS.md`, `.kiro/index.md`
```

## Future AI Guidance

Use this file for documentation and process breaking changes as well as code or data changes. Link migration steps clearly.

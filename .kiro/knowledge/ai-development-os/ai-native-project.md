# AI-Native Project

> This concept is generic and may later become part of the standalone AI Development OS project.

An AI-native project is structured so AI agents can understand and operate within it responsibly.

This does not require a specific programming language, framework, editor, or AI vendor. It requires clear project knowledge and repeatable engineering practices.

## Characteristics

- Agent instructions are explicit.
- Architecture rules are discoverable.
- Decisions are recorded.
- Domain knowledge is maintained.
- Workflows are repeatable.
- Quality gates are written down.
- Lessons learned become project memory.

## Human Responsibility

AI-native does not mean AI-owned.

Humans remain responsible for goals, judgment, review, safety, ethics, and long-term direction. AI agents assist by applying project knowledge consistently.
